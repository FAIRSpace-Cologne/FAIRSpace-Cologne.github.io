User Guidelines

Welcome to the FAIRdata Cologne User Guidelines. This comprehensive guide provides step-by-step instructions on how to use the FAIRdata Cologne platform effectively.

Getting Started

What is FAIRdata Cologne?

FAIRdata Cologne is a comprehensive platform designed to help researchers and institutions implement FAIR (Findable, Accessible, Interoperable, Reusable) data management principles. Our platform provides tools for creating, organizing, publishing, and discovering research data with full compliance to international standards.

Core Mission

We aim to facilitate FAIR data practices across research institutions and help researchers make their data discoverable, accessible, and reusable by the scientific community.

Platform Overview

The FAIRdata Cologne platform consists of several integrated modules:

Account Management Module

Create and manage user accounts with granular permission controls. Support for institutional authentication and single sign-on integration.

Metadata Editor

Create and edit metadata records with support for multiple metadata standards. The intuitive editor guides users through all required and optional fields.

Publication System

Publish and manage versions of metadata records. Track publication history and enable collaborative review workflows.

Discovery Portal

Search and browse published records with advanced filtering and faceted navigation capabilities.

System Requirements

Minimum Requirements

To use FAIRdata Cologne, you need the following minimum setup:

Browser Requirements

A modern web browser is essential. Supported versions include Chrome 90+, Firefox 88+, Safari 14+, and Edge 90+.

Network and Settings

A stable internet connection is required. JavaScript must be enabled in your browser, and cookies must be allowed for session management and user preferences.

Recommended Setup

For optimal performance and user experience, we recommend:

Hardware

Use a desktop or laptop computer with at least 4GB of RAM. Mobile devices can be used for browsing, but editing operations work best on larger screens.

Display

Screen resolution of 1366x768 or higher is recommended for comfortable use of all platform features.

Connectivity

A high-speed internet connection (10+ Mbps) ensures smooth operation, especially when uploading files or working with large metadata records.

Access and Permissions

User Roles and Permissions

FAIRdata Cologne supports multiple user roles with different levels of access and capabilities:

Visitor Role

Visitors can browse and search published metadata records without requiring a login. This role is ideal for the general public and external researchers.

Contributor Role

Contributors can create, edit, and manage their own metadata records. They can draft records and submit them for publication review.

Publisher Role

Publishers can review, approve, and publish metadata records submitted by contributors. They ensure records meet institutional standards before publication.

Administrator Role

Administrators have full system access, managing users, settings, metadata standards, and system configuration.

Permission Matrix

Permission Details by Role

Each role has specific permissions for different operations. Visitors have read-only access, Contributors can create and edit, Publishers can review and approve, and Administrators have unrestricted access. For detailed information about specific permissions for each user role, please refer to the User Account Management guide.

Permission Inheritance

Higher-level roles inherit permissions from lower levels. For example, Publishers can perform all Contributor operations plus publishing functions.

Key Features Overview

Metadata Creation and Management

Comprehensive Metadata Support

Create comprehensive metadata records with support for multiple metadata standards including Dublin Core, MIAPPE, and custom standards. The intuitive editor guides you through all required and optional fields.

Template-Based Editing

Use predefined templates to ensure consistency and completeness of your metadata records. Templates can be customized for specific research domains.

Validation and Quality Assurance

Automatic Validation

Automatic validation ensures your metadata meets FAIR principles and selected standards before publication. Real-time feedback helps you improve data quality as you work.

Quality Checks

The system performs comprehensive quality checks including schema validation, controlled vocabulary verification, and completeness assessment.

Search and Discovery

Advanced Search

Advanced search capabilities with filters, full-text search, and faceted navigation make it easy to discover and access relevant metadata records.

Search Features

Search by keywords, author, date, subject area, and other custom fields. Save frequently used search queries for quick access.

Version Control

Full Version History

Track changes to metadata records with full version history. Easily revert to previous versions or compare different versions of a record side-by-side.

Change Tracking

See who made what changes and when, with detailed changelogs for audit and compliance purposes.

Support and Help

Getting Help

Self-Service Resources

Check the FAQ section for common questions. Review the step-by-step guides for specific tasks. Browse video tutorials for visual learning.

Contact Support

Common Issues and Troubleshooting

Login Problems

If you cannot log in, verify your internet connection, check that caps lock is off, and try resetting your password using the "Forgot Password" link.

Browser Compatibility

If you experience display issues, try a different browser or clear your browser cache. Ensure you are using a supported browser version.

Reporting Issues

Bug Reports

Found a bug? Please report it through our issue tracking system. Navigate to Help → Report an Issue or email us at feedback@fairspace-cologne.de.

Feature Requests

Have a feature request? We welcome suggestions from our users. Submit your ideas through the feedback portal or during our community meetings.
